INSTALLATION

1. Установить Apache Thrift: http://thrift.apache.org/
2. Выполнить "mvn clean install" в директории "authservice"

Для юнит тестирования используется HSQLDB. Т.е. для получения артефакта можно 
не настраивать никаких СУБД.

Артефакт собирается со всеми зависимостями. Для запуска приложения сначала 
необходимо настроить MySQL. Для этого нужно создать пользователя coltrane 
(по умолчанию без пароля, это настраивается), базу данных coltrane_db и дать 
юзеру гранты на все операции.

CREATE USER coltrane;
CREATE DATABASE coltrane_db;
USE coltrane_db;
GRANT ALL PRIVILEGES ON *.* TO coltrane@localhost;

Дальше достаточно выполнить в консоли java -jar AuthServiceImpl-${version}.jar.

После запуска все таблицы удаляются и создаются новые. На этапе разработки 
так удобнее (как по мне). Это настраивается.

P.S. могу захреначить такую фишку, чтобы можно было переключаться на разные 
СУБД при запуске приложения. Например, в пропертях выбираешь HSQLDB или MySQL и 
и приложение будет работать с соответствующей СУБД. Плюс в том, что можно будет ваще 
ниче не настраивать, чтобы поднять сервис.

P.P.S. сейчас Persistence-Unit для юнит тестирования объявляется в стандартном persistence.xml. 
Отдельный для тестирования создать не получилось из-за баги в мавен-компайлере.
